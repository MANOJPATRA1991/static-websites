Gaussian Mixture Modelling (GMM)

What are Gaussian Mixture Models?

It is a type of Soft clustering.

Gaussian Mixture Models (GMMs) assume that there are a certain number of Gaussian distributions, and each of these distributions represent a cluster. Hence, a Gaussian Mixture Model tends to group the data points belonging to a single distribution together.

For a given set of data points, the GMM would identify the probability of each data point belonging to each of the available Gaussian distributions.

Gaussian Mixture Models are probabilistic models and use the soft clustering approach for distributing the points in different clusters.

What is Gaussian distribution (or Normal distribution)?

It has a bell-shaped curve, with the data points symmetrically distributed around the mean value.
Each Gaussian distribution has a mean μ\muμ and variance σ2\sigma^2σ2

How is the mean and variance value for each Gaussian assigned?

Expectation-Maximization

Expectation-Maximization (EM) is a statistical algorithm for finding the right model parameters. We typically use EM when the data has missing values (latent variables), or in other words, when the data is incomplete.

Step 1:

E-step - the available data is used to estimate the values of the latent variables

Step 2:

M-step - based on the values calculate in the E-step, the complete data is used to calculate the model parameters

How is k-means different from GMM?

k-means only considers the mean to update the centroid while GMM takes into account the mean as well as the variance of the data.

Advantages of GMM

  1. Soft clustering
  2. Cluster-shape flexibility

Disadvantages of GMM

  1. Sensitive to initialization values
  2. Possibility of convergence to a local minimum
  3. Slow convergence rate

Applications of GMM

  1. Non-parametric discovery of human routines from sensor data
  2. Application of the Gaussian mixture model in pulsar astronomy
  3. Biometrics - Speaker verification using Adapted Gaussian mixture models
  4. Adaptive background mixture models for real-time
